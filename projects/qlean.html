Qlean Cleaner

1

Клинер без приложения

Qlean — сервис уборки. Клиент заказывает через сайт, менеджер распределяет заказы. Но клинер — исполнитель — работает вслепую.

Звонок от менеджера. Адрес в WhatsApp. Чеклист — на бумаге или в голове. Фотоотчёт — скинуть в общий чат. Заработок — спросить в конце месяца.

Инструмент, который видит только клинер. Для людей, которые делают работу руками.

Задача

Спроектировать мобильное приложение для клинера офисных помещений. Три сценария:

  • Принять заказ — увидеть адрес, время, объём, контакт менеджера
  • Выполнить работу — отметить каждый пункт чеклиста, приложить фото
  • Получить деньги — видеть баланс, историю выплат, бонусы

Ограничения: экран всегда мокрый или в перчатках. Крупные зоны касания. Минимум текста. Один палец.

5

UX для мокрых рук

Клинер работает в перчатках. Экран часто мокрый. Палец один — обычно большой.

  • Чеклист — один тап. Нажал — отметил. Никаких свайпов, long press, подтверждений.
  • Зоны касания 48×48. Минимум WCAG, но для перчаток лучше 56px.
  • Кнопка «Следующий пункт» вместо скролла к незаполненному пункту.
  • Фото без галереи. Камера открывается сразу. Снял — загрузилось.
  • Минимум текста. Адрес, время, сумма. Описание — по тапу.
6

Дизайн-система

Палитра: Cyan #0891B2 + фон #F0FDFA. Ассоциация — чистота, вода, свежесть. Зелёный #22C55E для денег и завершённых задач.

Типографика: Inter. Единый шрифт для всех размеров. Разграничение — через вес (400/600/700), не через стиль.

Карточки: Белые, 14px radius, минимальная тень. Содержат ровно одну сущность — один заказ, один пункт чеклиста, одну выплату.

Навигация: Четыре таба — Заказы, График, Деньги, Ещё. Нет гамбургеров, нет вложенности.

7

Процесс

  • Исследование: интервью с тремя клинерами (WhatsApp), наблюдение за рабочим днём
  • User flow: три сценария — принять, выполнить, получить
  • Wireframes: бумага, 12 вариантов чеклиста
  • UI: Figma, 6 экранов, 2 итерации
  • Прототип: интерактивный Figma-прототип для тестирования
