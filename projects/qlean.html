Клинер без приложения

Qlean — сервис уборки офисов. Клиент заказывает через сайт, менеджер распределяет заказы. Но у клинера — исполнителя — не было никакого инструмента.

Вся коммуникация шла через колл-центр: адрес заказа, время, чеклист, отчёт — всё по телефону. КЦ был перегружен, клинеры работали вслепую.

Задача — дать клинеру приложение. Заказы, таймер, чеклист, выплаты — всё в одном месте. Разгрузить КЦ и дать исполнителю автономность.

Задача

Спроектировать мобильное приложение для клинера офисных помещений. Три сценария:

  • Принять заказ — увидеть адрес, время, объём, контакт менеджера
  • Выполнить работу — отметить каждый пункт чеклиста, отследить время
  • Получить деньги — видеть баланс, историю выплат, разбивку по дням

Ограничения: экран всегда мокрый или в перчатках. Крупные зоны касания. Минимум текста. Один палец.

UX для мокрых рук

Клинер работает в перчатках. Экран часто мокрый. Палец один — обычно большой.

  • Чеклист — один тап. Нажал — отметил. Никаких свайпов, long press, подтверждений.
  • Зоны касания 48×48. Минимум WCAG, но для перчаток лучше 56px.
  • Кнопка «Выезжаю на уборку» запускает таймер. Один тап — смена состояния.
  • Фото без галереи. Камера открывается сразу. Снял — загрузилось.
  • Минимум текста. Адрес, время, сумма. Описание — по тапу.
